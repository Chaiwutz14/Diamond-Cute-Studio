 Diamond Cute Studio 💎 — โฟลเดอร์ /data (Static Snapshot)
═══════════════════════════════════════════════════════════

โฟลเดอร์นี้ใช้เก็บ "ภาพรวมข้อมูลสินค้า/แกลเลอรี" เป็นไฟล์ JSON
เพื่อให้หน้าเว็บอ่านจากไฟล์ (ผ่าน CDN ของ GitHub Pages) แทนการอ่าน
Firestore โดยตรง → รองรับผู้ใช้ได้ "ไม่จำกัด" โดยไม่เปลืองโควต้าฟรี

──────────────────────────────────────────
 ไฟล์ที่ระบบจะอ่าน (ต้องสร้างเอง):
──────────────────────────────────────────
  • products.json   ← รายการสินค้า (active)
  • gallery.json    ← รูปผลงานในแกลเลอรี

  ❗ ตอนนี้ยังไม่มี 2 ไฟล์นี้ — ระบบจะ fallback ไปอ่าน Firestore ให้เอง
     (เว็บทำงานปกติ ไม่พัง) จนกว่าจะสร้างไฟล์ขึ้นมา

  ไฟล์ *.sample.json คือ"ตัวอย่างรูปแบบ" เท่านั้น ระบบไม่อ่าน

──────────────────────────────────────────
 วิธีสร้าง/อัปเดต (ง่ายมาก ทำผ่านเว็บล้วน):
──────────────────────────────────────────
  1) เปิดไฟล์  tools/export-snapshot.html  บนเบราว์เซอร์
     (เช่น https://chaiwutz14.github.io/Diamond-Cute-Studio/tools/export-snapshot.html)
  2) กดปุ่ม "สร้าง Snapshot" → ระบบจะดาวน์โหลด products.json และ gallery.json
  3) อัป 2 ไฟล์นี้ขึ้น GitHub ในโฟลเดอร์ /data/ (ผ่านหน้าเว็บ GitHub.com
     → กด Add file → Upload files → ลากไฟล์ลง → Commit)
  4) เสร็จ! หน้าเว็บจะอ่านจากไฟล์ทันที (อ่าน Firestore = 0)

  ทำซ้ำขั้นตอนนี้ "ทุกครั้งที่เพิ่ม/แก้/ลบสินค้าหรือแกลเลอรี"
  เพื่อให้ไฟล์ snapshot ตรงกับข้อมูลจริง

──────────────────────────────────────────
 ปิดระบบ snapshot (กลับไปอ่าน Firestore ตรง):
──────────────────────────────────────────
  ตั้ง  USE_SNAPSHOT: false  ใน js/config.js

──────────────────────────────────────────
 รูปแบบไฟล์ JSON:
──────────────────────────────────────────
  { "generatedAt": "...", "count": N, "items": [ {...}, {...} ] }
  ดูตัวอย่างจริงใน products.sample.json / gallery.sample.json
