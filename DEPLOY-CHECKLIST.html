1. วาง Firestore Rules ตัวใหม่

SEC-01

เปิดไฟล์ firebase-rules/firestore.rules ก๊อปทั้งหมด แล้ววางทับใน Console

Firebase Console → Firestore Database → แท็บ Rules → ลบของเดิม วางของใหม่ → Publish

อีเมลแอดมินใส่ให้แล้ว (peeza1482546@gmail.com) · รวม slipGuard + ปิดช่องส่งยอด ฿0 · ตอนนี้เหลือไฟล์ rules ไฟล์เดียว (ลบ legacy/v15-secure ที่ซ้ำซ้อนออกแล้ว)

2. ตั้ง ALLOWED_ORIGIN + อัปโค้ด Worker ใหม่

SEC-03

เอาโค้ดใหม่จาก cloudflare-worker/index.js วางใน Worker แล้ว Deploy (ตอนนี้ค่าเริ่มต้นกันคนนอกให้แล้ว แต่ตั้ง secret ให้ชัวร์ขึ้น)

  1. Cloudflare Dashboard → Workers & Pages → เลือก Worker ของร้าน
  2. แท็บ Settings → Variables and Secrets → Add: ชื่อ ALLOWED_ORIGIN ค่า https://chaiwutz14.github.io
  3. แท็บ Edit code → วางโค้ดใหม่ทั้งไฟล์ → Deploy

เดิม Worker เปิดให้ทุกเว็บยิง /upload, /notify ได้ · ตอนนี้รับเฉพาะโดเมนร้าน

3. เปิด Firebase Storage + วาง storage.rules

SEC-02

ทำให้สลิปโอนเงินเป็นแบบ private (อ่านได้เฉพาะแอดมิน)

  1. Firebase Console → Storage → Get started (เลือก location ใกล้ไทย เช่น asia-southeast1) → เปิดใช้
  2. แท็บ Rules → วางทับด้วยไฟล์ firebase-rules/storage.rules → Publish
✨

ใน config.js ตั้ง PRIVATE_UPLOADS: true ให้แล้ว — พอเปิด Storage เสร็จ สลิปจะกลายเป็น private อัตโนมัติทันที ถ้ายังไม่เปิด Storage ระบบจะ fallback ไป ImgBB ให้เอง (ออเดอร์ไม่พัง)

P1

ทำเร็วๆ นี้

4. สร้าง + อัป Static Snapshot (สำคัญต่อการรับคนเยอะ)

PERF

ขั้นตอนนี้คือหัวใจของการ "รับ 100 คน/ชม. โดยไม่เปลืองโควต้า Firestore"

  1. เปิด tools/export-snapshot.html บนเบราว์เซอร์ (อัปขึ้น GitHub ก่อนแล้วเปิดผ่านลิงก์ หรือเปิดไฟล์ในเครื่องก็ได้)
  2. กดปุ่ม "สร้าง Snapshot" → ได้ไฟล์ products.json + gallery.json
  3. อัป 2 ไฟล์ขึ้น GitHub ในโฟลเดอร์ data/ (Add file → Upload files → Commit)

หน้าบ้านจะอ่านจากไฟล์ (ผ่าน CDN) แทน Firestore → ลดการอ่านจาก ~167 เหลือ ~0 ต่อการเข้าชม · ทำซ้ำทุกครั้งที่แก้สินค้า

เสริม

ทำเมื่อพร้อม (ระบบทำงานได้แม้ยังไม่ทำ)

5. เปิด Enforce App Check

SEC-08

กันบอตยิง Firestore/Auth ตรงด้วย config สาธารณะ (site key ตั้งไว้ในโค้ดแล้ว เหลือแค่กด Enforce)

Firebase Console → App Check → เลือก Firestore และ Authentication → Enforce

6. สร้าง Composite Index ของรีวิว

BUG-04

ทำให้รีวิวต่อสินค้าโหลดเร็วขึ้น (แก้โค้ดให้ทำงานได้แม้ไม่มี index แล้ว — index แค่ทำให้เร็วขึ้น)

  1. เปิดหน้า product ของสินค้าที่มีรีวิว → เปิด Console เบราว์เซอร์ (F12)
  2. ถ้ามีลิงก์ "create index" ขึ้นมา → กดลิงก์ → Create (รอ 1-2 นาที)

7. ตรวจรหัสแอดมิน + ลบเส้นทางรหัสสำรอง (ถ้าต้องการ)

SEC-06

ตอนนี้ใช้ Firebase Auth จริงแล้ว — ตั้งรหัสผ่านแอดมินให้ยาว/ซับซ้อน และถ้าต้องการความปลอดภัยสูงสุด สามารถลบเส้นทาง login สำรอง (hash) ใน admin-core.js ออกได้ (ไม่บังคับ)
