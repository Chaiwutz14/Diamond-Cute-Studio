# 📖 คู่มือผู้ใช้งานหลังบ้าน — Diamond Cute Studio
*สำหรับเจ้าของร้าน/แอดมิน (ไม่ต้องมีความรู้โค้ด)*

## 🔑 เข้าหลังบ้าน
1. เปิด `https://chaiwutz14.github.io/Diamond-Cute-Studio/admin-login.html`
2. ใส่รหัสผ่าน → เข้าสู่ระบบ (เครื่องเดิมจะจำไว้ ไม่ต้องพิมพ์ซ้ำ)
3. ออกจากระบบ: ปุ่ม "ออก" มุมขวาบน

## 🛍️ จัดการสินค้า
**เพิ่มสินค้า:** จัดการสินค้า → + เพิ่มสินค้า → กรอกชื่อ/ราคา/หมวด → รูป: กด "📤 อัปโหลดรูป" (เพิ่มได้หลายรูป, กด "☆ ตั้งเป็นรูปปก" เลือกรูปหลัก, ใส่ "ชื่อแบบ" ให้ตรงตัวเลือกขนาดเพื่อให้รูปสลับตามที่ลูกค้ากด) → วิดีโอ: วางลิงก์ YouTube/TikTok → บันทึก
**เพิ่มหมวดหมู่ใหม่:** ใน dropdown หมวดหมู่ เลือก "➕ เพิ่มหมวดหมู่ใหม่..." → ตั้งชื่อ+อิโมจิ → ใช้ได้ทั้งเว็บทันที
**เปิดให้ลูกค้าลองใส่รูป (Preview):** ติ๊ก "เปิดให้ลูกค้าลองดูตัวอย่าง" ในฟอร์มสินค้า

## 📦 จัดการออเดอร์
ออเดอร์ทั้งหมด → กดดูรายละเอียด → เปลี่ยนสถานะ: รับออเดอร์ → จัดเตรียม → **กำลังส่ง (ใส่ขนส่ง+เลขพัสดุ)** → ส่งสำเร็จ → บันทึก
ลูกค้าเห็นสถานะ+กดตรวจพัสดุเองได้ที่หน้า "ติดตามออเดอร์"

## ⭐ จัดการรีวิว
รีวิวลูกค้าจะรอใน "⏳ รออนุมัติ" → กด ✅ อนุมัติ จึงแสดงบนเว็บ · เพิ่มรีวิวจากร้านเองได้ปุ่ม "+ เพิ่มรีวิวจากร้าน"

## 📝 แก้เนื้อหาเว็บ (ไม่ต้องแตะโค้ด)
เมนู "เนื้อหาเว็บไซต์" แก้ได้: ข้อความหน้าแรก, แบนเนอร์โปร, ตัวเลขความเชื่อมั่น, **ช่องทางติดต่อทุกช่อง (LINE/FB/IG/TikTok/อีเมล)**, **PromptPay (เปลี่ยนบัญชีที่นี่)**, คำถามที่พบบ่อย → กด "💾 บันทึกทั้งหมด" มีผลทันที
> 💡 ช่อง "ชื่อ LINE ID" (@xxx) สำคัญ — ทำให้ปุ่ม LINE ทุกจุดเปิดแชทพร้อมข้อความอัตโนมัติ

## 🖼️ ตัวอย่างงาน & 🎨 เทมเพลต
ตัวอย่างงาน: อัปรูปผลงานจริง → แสดงหน้า "ตัวอย่างงาน" + หมุนบนหน้าแรก
เทมเพลต: อัปกรอบ PNG (กลางโปร่งใส เช่นทำจาก Canva) ให้ลูกค้าลองใส่รูป

## 💾 สำรองข้อมูล (สำคัญมาก!)
ตั้งค่า → "💾 สำรองข้อมูล" → **⬇️ Export ข้อมูลทั้งหมด** ได้ไฟล์ .json เก็บไว้
แนะนำ: ทำทุกสัปดาห์ · กู้คืน: ⬆️ Import แล้วเลือกไฟล์ backup

## 🔐 เปลี่ยนรหัสผ่าน
- **แบบ Firebase (แนะนำ):** Firebase Console → Authentication → Users → จุดสามจุดหลังบัญชีแอดมิน → Reset password
- **แบบเดิม (hash):** ตั้งค่า → สร้าง Hash → copy ไปแทน `ADMIN_HASH` ใน js/admin.js → อัปไฟล์ขึ้น GitHub

## ⬆️ อัปเดตไฟล์เว็บ
github.com → repo Diamond-Cute-Studio → Add file → Upload files → ลากไฟล์ → **พิมพ์ commit message สั้นๆ ว่าแก้อะไร** (เช่น "v15: เพิ่มระบบ backup") → Commit → รอ 1-2 นาที เว็บอัปเดต

## 🔎 V24 — ข้อควรรู้สำหรับแอดมิน

### ตรวจ "ยอดเงินในสลิป" เองทุกครั้ง ⚠️
ระบบตรวจสลิปอัตโนมัติ (โหมดฟรี) เช็กแค่ว่า **"เป็นสลิปจริง + ไม่เคยใช้ซ้ำ"** เท่านั้น
**ไม่ได้เช็กว่ายอดเงินตรงกับออเดอร์** และไม่บล็อกการสั่งซื้อ (ติดธงเตือนให้แอดมินดูเอง)
→ ก่อนกดจัดส่ง/ผลิต **ให้เทียบยอดในสลิปกับยอดออเดอร์ทุกครั้ง** โดยเฉพาะออเดอร์ที่มีธง ⚠️ สีเหลือง
(ถ้าต้องการตรวจยอดอัตโนมัติ ต้องเปิด API ตรวจสลิป — มีค่าใช้จ่ายตามผู้ให้บริการ ดู HANDOVER.md)

### เผยแพร่ snapshot หลังแก้สินค้า/ราคา
เมื่อเพิ่ม/แก้/ลบสินค้า ราคา หมวด หรือรีวิว — **ต้องกด "เผยแพร่ snapshot"** หน้าเว็บลูกค้าถึงจะอัปเดต
- แถบเผยแพร่จะบอก **"🕓 เผยแพร่ล่าสุดเมื่อ..."** ให้เช็กว่าตรงกับการแก้ล่าสุดหรือยัง
- Footer หน้าเว็บลูกค้าจะแสดง "🗓️ อัปเดตแคตตาล็อก..." ด้วย
- ถ้าลูกค้าเห็นราคาเก่าในตะกร้า ระบบจะ **เตือน + มีปุ่ม "อัปเดตราคา"** ให้ลูกค้าเองอัตโนมัติ
